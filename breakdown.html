01 · THE CORE

One camera, two hands, zero clicks

A single getUserMedia feed runs MediaPipe Hands (2 hands, modelComplexity 1). Left/right is decided by on-screen X after mirroring — not MediaPipe's handedness label. A shared gesture state (point · grab · pinch · open) drives every app, so there's only ever one camera and one tracker.

THE GESTURE VOCABULARY
  • Pinch + drag → spin the app carousel
  • Point + dwell → open / play (a ring charges so it never misfires)
  • ✊ Hold a fist → exit an app
  • Open palm → enter from the loading screen

03 · THE GESTURE-FX PROMPT

The spec that started the rig

Gesture FX began as one detailed brief — a CDJ-style, two-handed audio-FX rig. The shape of that prompt:

Prompt · core idea
Build a CDJ-style hand-gesture audio FX rig using MediaPipe Hands + Tone.js. Track both hands; decide left/right by on-screen X after mirroring. Effects sit on parallel sends (wet=1), controlled by per-effect send-gains: filter (HP/LP isolator) · echo · delay · reverb · flanger · roll Map: right-Y → filter · right openness → delay · left-Y → reverb left openness → echo · hand distance → flanger · both fists → roll CRITICAL: when a hand leaves frame, hard-gate its targets to neutral/0 so the FX don't blast when nothing's in view.
HARD-WON LESSONS
  • Canvas must set its CSS size to the viewport, or DPR offsets every drawing by 2×.
  • Map landmarks object-cover-correct so the skeleton lands on the webcam.
  • Effects belong on the unfiltered signal — otherwise the filter silences them.
  • YouTube FX needs a tab-audio share permission (browsers can't read an iframe's audio) — built-in tracks are the frictionless path.
04 · RUN IT YOURSELF

Setup in 3 commands

Terminal
git clone https://github.com/sharonxu-wav/blank-space.git cd blank-space npm install npm run dev # open the printed localhost URL, allow camera

Needs Chrome or Edge and a webcam. Build for production with npm run build (output in dist/). Tip: ?windows swaps the carousel for a draggable floating-window menu.
